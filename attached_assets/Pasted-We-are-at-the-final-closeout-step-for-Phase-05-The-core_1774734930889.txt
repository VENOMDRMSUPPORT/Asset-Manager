# Required layout direction

## 1) Left side: operator-facing execution surface
The left-side experience should feel more like an Asset Manager / execution console than a plain passive file list.

This does NOT mean deleting the file explorer completely.
It means the left-side experience should better communicate:

- what the agent is doing
- what files it touched
- what stage it is in
- what actions it completed
- what the final result was

The user should feel that someone is actively working, not that they are staring at a static file tree.

Possible implementation direction:
- preserve file explorer access
- but give stronger visual/operator emphasis to execution feed and action grouping
- improve grouping of execution steps and summaries
- make the execution surface feel like a real working session

## 2) Right side: professional task context panel
The right-side panel should feel more intentional and professional.

Requirements:
- live-updating task list
- clearer current task vs selected historical task
- better summaries for completed tasks
- stronger context for the selected task
- if useful, show:
  - files touched
  - commands run
  - verification result
  - repair count
- do not just duplicate the entire file explorer for looks

The right side should act as a strong task context / task management surface.

## 3) Execution Feed should become more central
The current execution feed exists, but it still feels secondary.

Improve it so that:
- it is visually more central to the working experience
- execution stages are easier to scan
- grouped actions feel more professional
- completion summaries are easier to trust
- the user can understand “what happened” without hunting for it

## 4) Better professional completion summaries
When a task finishes, the summary should feel closer to a professional execution tool.

Include clean, readable sections such as:
- files changed
- commands run
- verification outcome
- repair attempts if any
- final result

Do not make it verbose or fake.
Do make it feel polished and grounded.

## 5) Preserve what already works
Do not regress:
- task execution
- verification
- repair loop
- task history persistence
- task replay
- VenomGPT naming
- project intelligence
- stage-aware feed
- right-side task usability that already exists

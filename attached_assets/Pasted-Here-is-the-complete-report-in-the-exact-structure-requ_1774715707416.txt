2. Why the bug happened
The old parseAction() function extracted from the first { to the last } in the response. When the model returned multiple JSON objects:

{"action":"think","thought":"..."}
{"action":"write_file","path":"app.js","content":"..."}
This produced a string that starts at the first { and ends at the last } — a two-object concatenation that JSON.parse() correctly rejects. This caused the "model returned invalid responses repeatedly" failure even when the model output was logically correct.

For conversational prompts like hi, the model would return readable prose. The old loop counted this as a parse failure and would retry up to 3 times before failing hard — even though there was nothing wrong.

3. How parsing / normalization / repair now work
New 5-step normalization pipeline in responseNormalizer.ts:

Step	What it does
0	Classify response as conversational if no { found and text matches greeting patterns — classified, not a parse failure
1	Direct parse attempt (fast path for perfect model output)
2	Strip markdown code fences (```json ... ```) and retry parse
3	Extract all syntactically complete JSON objects using a character-by-character brace-depth scanner
4	If exactly one action object found, accept it
5	If multiple action objects found, pick best by priority: done > write_file > run_command > read_file > list_dir > think
Safety invariants:

Never invents actions
Malformed or ambiguous output still returns a typed failure (NormalizeFailure)
All failures have a specific reason: conversational, no_json, malformed, no_action_field, ambiguous
Retry instructions are now specific per failure reason. buildRetryInstruction() produces a message that states exactly what went wrong, not a generic "your JSON was bad."

Normalization method is logged (direct_parse, fence_stripped, first_extracted, best_selected) so every recovery is visible in debug logs. Warnings are surfaced when multiple objects were found.

4. How non-task prompts are handled now
Before entering the agent loop, agentLoop.ts checks isConversationalPrompt():

Prompt: "hi" → detected as conversational
  → Single chat() call with a simple system prompt
  → Model replies naturally (e.g. "Hello! I'm DevMind. How can I help?")
  → Emits as thought event, completes with status "done"
  → Uses GLM-4.7-Flash (free tier, fast, no heavy reasoning needed)
  → No parse loop, no retry, no failure
Observed live: hi completed in 2.2 seconds with a clean done status and accurate durationMs — no parse failure, no retry noise.

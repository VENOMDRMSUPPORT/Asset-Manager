For screenshot-driven tasks, VenomGPT must become much stronger at:

* identifying visible UI defects

This phase should substantially improve analysis quality for:

* spacing and alignment issues

* incorrect text / labels / hierarchy

### D) Honest visual execution path

If visual analysis succeeds:

* use the visual findings to guide code inspection and edits

* fail honestly or degrade only in an explicitly truthful, controlled way

Implement the phase so that:

* multimodal task intake is not hardcoded as “Z.AI forever”

* screenshot task classification
* visual analysis orchestration

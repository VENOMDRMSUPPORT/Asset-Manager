- The current Execution Feed does not yet match the Replit-style feed structure, density, icon usage, row rhythm, and activity presentation closely enough.

## Implementation requirements
### 1) Layout hierarchy must follow Replit
Implement the workspace using a Replit-like multi-panel shell:
- Left explorer rail/panel
- Large center workspace
- Narrow right task history rail
- Bottom execution/terminal section attached to the center workspace

The center area must clearly be the dominant region.

### 2) Right Task History rail sizing
Correct the right rail so it behaves like a narrow secondary column.
Target behavior:
- It should be noticeably narrower than it is now.
- It should hold compact task cards with truncation and controlled wrapping.
- It should not consume space intended for the main workspace.
- It should maintain usability without becoming the visual focus.

Use a width model closer to a fixed or tightly clamped rail rather than an aggressively flexible panel.
Prefer something in the spirit of a narrow rail, not a full content pane.

### 3) Center workspace dominance
The center workspace must expand naturally and retain maximum usable width.

The bottom panel must be treated as an execution/activity surface, not a generic output box.

- compact dark panel body
- feed items stacked vertically with tight spacing
- action groups that look like operational steps
- compact inline metadata where applicable
- consistent row padding and readable density
- execution-first presentation rather than conversational presentation

When a task is submitted, the resulting items in the Execution Feed should resemble Replit’s activity cards/rows as closely as possible:
- compact blocks
- small icon chips/buttons for actions/tools
- grouped action badges

- subtle borders/background separation
- strong density control

- no bloated padding

The feed should support patterns similar to:

- action/tool block
